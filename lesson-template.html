How Python Stores Data in Memory

Variables are labels, not boxes — and that distinction changes everything.

The Core Idea

In Python, a variable is not a box that holds a value. It is a name attached to an object that lives in memory. When you write x = 5, Python creates an integer object, then binds the name x to it.

This distinction matters the moment two names point to the same object. The object exists once — the names are merely different ways to reach it.

Definition 3.1 — Name Binding

A name binding is an association between an identifier and a Python object. Multiple names may bind to the same object simultaneously. Rebinding a name changes only that association — it does not alter the object itself.

Code

Python · variable binding Listing 3.1 
# Two names, one object
x = 42
y = x

print(id(x) == id(y))  # True — same object in memory

# Rebinding x doesn't move y
x = 100
print(y)  # still 42

Equation

Equation 3.1 — Identity equality
\[ \text{id}(x) = \text{id}(y) \iff x \text{ and } y \text{ reference the same object} \] (3.1)
Python's id() returns the memory address of the object a name currently points to.

Key Points to Remember

  • 01 Python variables are names bound to objects — not containers holding values.
  • 02 id(x) returns the memory address of the object x currently points to.
  • 03 Writing y = x makes y point to the same object — not a copy of it.
  • 04 Rebinding one name never affects other names pointing to the same object.
